Cluster-Cluster Aggregation on ℤ²

Noam Borgnia · Master's thesis under Prof. Noam Berger, TU Munich, 2026

The Model

We study a stochastic aggregation process on the two-dimensional integer lattice. Begin with particles scattered randomly across the grid — each site is independently occupied with probability p. Every particle starts as its own cluster of mass 1.

Clusters evolve in continuous time. Each cluster of mass m carries an independent exponential clock ringing at rate m−α, where α is a fixed parameter controlling how mobility depends on size. When a cluster's clock rings, it attempts to translate rigidly by one lattice step in a uniformly random direction. If the move causes overlap with another cluster, the two merge at their current positions; otherwise the cluster simply translates.

The parameter α controls three qualitatively distinct regimes. For α < 0 (explosive), large clusters move faster than small ones: a slight size advantage accelerates into runaway growth, and in infinite volume an infinite component forms in finite time. At α = 0 (mass-independent), all clusters diffuse at the same rate regardless of size — aggregation is fast and democratic. For α > 0 (physical), large clusters are nearly frozen while singletons dart around and get absorbed. Growth is hierarchical: clusters form a tessellation at each mass scale, and the next scale only becomes active on a much slower timescale. This gridlock mechanism produces the conjectured t1/α growth of the origin cluster.

The Gillespie Algorithm

The simulation is an exact implementation of the continuous-time Markov chain governing the cluster dynamics. Rather than ticking a global clock and checking whether each cluster acts, the Gillespie algorithm exploits the memoryless property of exponential distributions to jump directly from one event to the next.

At any moment, each cluster i of mass mi has an independent exponential clock at rate mi−α. By superposition, the next event across all clusters occurs at rate R = Σ mi−α, and the waiting time to that event is Exp(R). Conditioning on an event occurring, the probability that it was triggered by cluster i is mi−α / R.

Each step of the algorithm therefore proceeds as follows:

  1. Compute the total rate R = Σ mi−α over all clusters.
  2. Advance time by Δt ~ Exp(R).
  3. Select the active cluster A with probability proportional to its rate.
  4. Choose a uniformly random lattice direction and attempt the move-or-merge step.

This is exact: each run produces a valid sample path of the continuous-time process. The efficiency comes from collapsing all individual clocks into a single Poisson process, avoiding the need to track residual waiting times.
